Dataset Analysis & Key Findings

Explore the underlying distributions and relationships in the credit card transactions dataset. Since features V1–V28 are PCA-transformed components for customer privacy, we focus on identifying class separability patterns, transaction volumes, and skewness of untouched transaction attributes.

1. Skewed Transaction Amount Distribution

Insight: The raw transaction amount distribution is heavily right-skewed. To capture patterns effectively across linear models, a log-scaling mapping was applied. Note how fraud transactions (pink) exhibit different density profiles, often clustering around larger dollar amounts or specific micro-ranges.

2. Transaction Volume & Nocturnal Activity

Insight: The distribution of transactions across the 24-hour cycle. Fraud (pink) shows elevated activity relative to total transaction rates during nocturnal off-hours (between hours 2 and 6 AM) when legitimate transactions are at their lowest point, presenting a strong indicator.

3. Top Discriminative Features Heatmap

Insight: The correlation matrix highlights the components showing the strongest positive and negative linear correlation with our target Class. PCA features V17, V14, V12, and V10 exhibit strong negative correlations, while V4 and V11 correlate positively.

4. Class Separability (V17 vs V14)

Insight: Plotting the top two negatively-correlated PCA features (V17 vs V14) demonstrates how fraud transactions cluster cleanly in the bottom-left quadrant. This visual clustering validates that the dataset contains strong, separable features for machine learning models to capture.

Confusion Matrices Comparison

Side-by-side matrices showing the count of Legit and Fraud transactions detected correctly or missed.

Notice that while Logistic Regression catches slightly more fraud (Recall 87% vs 84%), it creates 1,617 False Alarms (Precision 5%), whereas XGBoost drastically reduces false alarms to 205 (Precision 28%) while maintaining an 84% detection rate.

Precision-Recall (PR) Curves

Illustrates the model precision relative to recall across all decision probability cutoffs.

XGBoost (green curve, PR-AUC = 0.7810) stays much higher towards the top-right than Logistic Regression (purple curve, PR-AUC = 0.6729), indicating that XGBoost yields superior detection accuracy and cleaner threshold tuning.
