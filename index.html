📌 Overview

Why This Project Matters

Standard classifiers fail on extreme imbalance. A naive model predicting "legit" for everything would achieve 99.83% accuracy — but catch zero frauds.

⚖️

SMOTE Oversampling

Synthesizes new minority-class (fraud) training examples by interpolating between existing fraud samples — without naive duplication.

🎯

PR-AUC Optimization

Optimizes for Precision-Recall AUC — not accuracy — to honestly measure real fraud-detection performance on imbalanced data.

🔬

RobustScaler

Transaction amounts span $0 to $25,000+. RobustScaler uses median/IQR instead of mean/std, resisting extreme outliers that distort normalization.

🌐

Live Web Dashboard

FastAPI + Glassmorphic UI with real-time transaction scoring, animated risk gauge, feature impact bars, and EDA chart gallery.

⚙️ Pipeline

ML Pipeline Architecture

A 7-stage pipeline from raw CSV to live API dashboard.

📂

Raw Data Ingestion

284,807 real-world European bank transactions. V1–V28 are PCA-derived features; Time and Amount are original.

creditcard.csv
🔧

Preprocessing

Remove 1,081 duplicate rows. Apply RobustScaler to Time & Amount. Stratified 80/20 train-test split.

src/preprocess.py
📊

Exploratory Data Analysis

Generate 6 publication-quality charts: amount dist, time density, correlation heatmap, scatter separability.

src/eda.py
⚖️

SMOTE Resampling

Oversample the minority fraud class in the training set only (never test set) to prevent data leakage.

src/train.py
🤖

Model Training

Train two classifiers in parallel: cost-balanced Logistic Regression and XGBoost with scale_pos_weight.

src/train.py
📏

Evaluate & Export

Calculate PR-AUC, ROC-AUC, F1, confusion matrices. Save models as .pkl and metrics as metrics.json.

models/metrics.json
🌐

FastAPI Web Dashboard

Serve real-time predictions, animated gauge visuals, model switching, and the EDA gallery on localhost:8000.

src/serve.py
